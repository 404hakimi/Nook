## 一、产品定位

**Nook 是一个独享 IP 代理服务的全套自动化系统**，最终形态是闲鱼小作坊级 SaaS：

- **卖什么**：会员买的是**一个独占 IP + 一段时长 + 一份流量额度**，从市面上常见的 99-299 元/月价位带切入
- **为啥独占 IP 值钱**：ChatGPT / Claude / 跨境电商工具对"住宅 IP / ISP IP / 干净 IP"敏感，纯共享 VPN 越来越用不动
- **谁是客户**：开 LLM 的开发者、跨境电商运营、远程办公租用海外服务者；下沉市场（闲鱼/淘宝）愿意为"专属 IP"多付一倍

## 二、技术总架构

### 物理拓扑（中转 + 落地）

```
[客户 v2rayN/Clash]
       │
       │ vmess (用户独占的 UUID + 套餐时长 + 套餐流量)
       ▼
[中转线路服务器 line-xxx]                  ← resource_server
   ├─ 纯 Xray 内核 (无 3x-ui, 由 nook 通过 gRPC 调度)
   ├─ 一个 vmess inbound (clients=[] 由 nook 在内存动态加)
   └─ Xray gRPC API on 127.0.0.1:62789  ← 经 SSH 隧道访问
       │
       │ nook 通过 gRPC (HandlerService.AlterInbound) 加 user
       │ nook 通过 gRPC (HandlerService.AddOutbound) 加 socks5 outbound
       │ nook 通过 gRPC (RoutingService.AddRule) 按 email 分流
       ▼
[落地节点 socks5-xxx (按地区 N 台)]        ← resource_ip_pool
   └─ 3proxy SOCKS5 server, 自购 VPS, 用户独占
       │
       ▼
   Internet (客户最终对外暴露的 IP)
```

### 关键设计决策（已确认）

| 决策点 | 选定 | 理由 |
|---|---|---|
| Xray 控制路径 | **方案 C: gRPC** | 用户体验优先 (无重启抖动), HandlerService.AlterInbound 等增量操作毫秒级 |
| 3x-ui 角色 | **不集成** | 运营若需可视化自己装一个; nook 不双写 3x-ui DB |
| SOCKS5 来源 | **自购 VPS 装 3proxy** | 独占 IP, 不依赖第三方 SOCKS5 商家 |
| nook ↔ Xray gRPC 通信 | **SSH 隧道转发** | gRPC 监听 127.0.0.1, 通过 SSH 端口转发到 nook |
| 部署方式 | **nook 携带模板脚本** | 后台一键部署, 参数 (port/UFW/BBR) 在表单里调; 仅支持 Ubuntu 22+ |
| 业务规则执行 | **nook 自己定时扫** | Xray 内核不管业务, nook 调度器到期/超量主动 revoke |

### 数据流：兑换业务核心

```
1. 用户买卡 → 拿到 CDK
2. 用户登 nook 前台兑换 CDK
3. nook 在事务中:
   ├─ 查 CDK (未使用 + 未过期)
   ├─ 锁定 CDK (Redis lock:cdk:{hash})
   ├─ 按套餐挑落地 IP (region+ip_type 过滤, status=available)
   ├─ 标 IP.status=occupied + assigned_member_id
   ├─ 创建 member_subscription (套餐 + 会员 + IP + expires_at + total_bytes)
   ├─ 调 XrayInboundService.provision:
   │    ├─ HandlerService.AlterInbound: 给中转的 vmess inbound 加 user
   │    │   (email=member_X_Y, uuid=随机, level=0)
   │    ├─ HandlerService.AddOutbound: 加 socks5 outbound
   │    │   (tag="user_X_Y_out", servers=[ip_pool 的 socks5 地址])
   │    └─ RoutingService.AddRule: 加路由规则
   │      (match: inboundTag=vmess-in & email=member_X_Y → outbound user_X_Y_out)
   ├─ 落 xray_inbound 行 (server_id, ip_id, member_user_id, client_uuid, client_email)
   ├─ CDK 标 used
   └─ 写 business_order_log
4. 前台展示订阅链接 vless://uuid@host:port?... 给用户复制到客户端
```

## 四、实施顺序（按优先级）

### Phase 1 · 中转控制核心（已完成基础设施 + 待补 gRPC）

- 服务器管理（CRUD + 一键部署 + Xray 状态/重启 + SSH 控制台）
- 部署脚本模板（Ubuntu 22+ Xray 安装，参数化 vmess port / gRPC port / UFW / BBR）
- nook ↔ Xray 抽象层（XrayBackend 接口 + Factory + 事件失效缓存）
- xray_inbound 元数据管理（manual provision、轮换、吊销、流量查询、本地编辑）

#### 1.2 待补 — gRPC 真正打通 ⏳ ≈ 4 天

**子任务**：

| 任务 | 工时 | 备注 |
|---|---|---|
| Vendor Xray 必要 proto 文件到 `nook-biz-xray/src/main/proto/` | 0.5 d | 约 10 个 proto, 含 typed_message / proxyman/command / router/command / stats/command / vmess account / socks config |
| Maven protobuf 插件 已配置, 跑 `mvn compile` 生成 Java stub | 0 d | 已配 |
| `XrayGrpcChannelManager`: 建 SSH 隧道 + Channel + 状态监听 | 1 d | sshj LocalPortForwarder + grpc Channel.notifyWhenStateChanged |
| `XrayGrpcBackend.addClient`: AlterInbound + AddOutbound + AddRule 三连 | 1 d | 业务关键路径 |

| `XrayGrpcBackend.getClientTraffic` / `resetClientTraffic`: StatsService | 0.5 d | |
| `XrayConfigReconciler`: channel 断重连后全量重新注入 + 周期对账 | 0.5 d | |

**完成标准**：管理后台手动 Provision 一条 inbound 后，从客户端 v2rayN 连到中转，看到的出网 IP = 落地 SOCKS5 的 IP。

### Phase 2 · 资源池模块 ⏳ ≈ 2 天

#### 2.1 resource_ip_type
- 三条种子 (ISP / 家宽 / 机房)
- 简单 CRUD（运营基本不动）

#### 2.2 resource_ip_pool
- entity + mapper（按 region/ip_type 索引）
- service + IP 状态机：`available → occupied → cooling → available`
- API 跨模块: `pickAvailable(region, ipTypeId, excludeIds)` 给兑换流用
- Controller + 前端 IP 池管理页（CRUD + 一键部署 SOCKS5 + 状态过滤 + 健康检查按钮）
- 一键部署用 `install-socks5-landing.sh.tmpl`，与线路服务器部署同样的模板渲染机制

#### 2.3 InboundProvisionDialog 升级
- `ipId` 从文本输入改为按 region+type 过滤的 IP 选择器（自动挑选状态=available 的）
- 选完后自动回填 IP 信息

### Phase 3 · 业务流（变现路径）⏳ ≈ 6 天

#### 3.1 member（会员）≈ 1 天
- entity + 注册登录（sa-token member 类型，与 system 用户隔离）
- 简化版前台 nook-portal（独立 SPA，复用 Select 等组件）
- 邮箱手机号二选一登录（先做邮箱，方便测试）

#### 3.2 business_plan（套餐）≈ 1 天
- 套餐 = 「allowedRegions + ipTypeId + durationDays + trafficGb + price + slots」
  - `slots` 默认 1，套餐买 N 个 IP 时为 N
- CRUD + 上下架开关
- 套餐与可分配资源的实时校验（套餐选了"美国 ISP"但池子里没货 → 标"暂时无货"）

#### 3.3 business_cdk（卡密）≈ 1 天
- 批量生成（运营选套餐 + 数量 → 生成 N 张 16 位卡密）
- 卡密状态机：未使用 / 已用 / 已过期 / 已禁用
- 导出 CSV（闲鱼发货用）
- 用 Hash 索引查询，密码学安全的随机生成

#### 3.4 business_recharge（兑换流，核心）≈ 1.5 天
- **核心事务**：CDK 校验 → IP 池挑选 → 创建 subscription → provision Xray → 标 CDK used
- Redis 锁防双卖：`lock:cdk:{hash}` + `lock:ip:{ipId}`
- 失败回滚补偿（Provision 失败要释放 IP 标 available + 撤销 subscription）
- 操作日志落 `business_order_log`

#### 3.5 member_subscription ≈ 0.5 天
- entity (member_id, plan_id, ip_id, xray_inbound_id, expires_at, total_bytes, used_bytes, status)
- 服务方法：拿当前订阅、续期（同套餐 CDK 累加 expires_at + 重置 traffic）
- 跨模块 API 给前台拉"我的订阅"

#### 3.6 nook-portal 会员前台 ≈ 1 天
- 路由：`/login` `/register` `/redeem` `/my/subscriptions` `/my/profile`
- 我的订阅卡片：二维码 + 复制配置链接（vless:// + 流量进度条 + 到期倒计时）
- 续费按钮（输入续费 CDK）

### Phase 4 · 自动化运营 ⏳ ≈ 3 天

#### 4.1 SubscriptionExpiryScheduler ≈ 0.5 天
- `@Scheduled(fixedDelay = 60_000)` 每分钟扫一次
- 到期：调 `XrayInboundService.revoke` + 标 subscription expired + 释放 IP（cooling 24h）
- 超量：同上

#### 4.2 流量统计 (monitor 模块) ≈ 1 天
- `monitor_traffic_snapshot`：5 分钟一次扫所有 active subscription，调 gRPC `StatsService.GetStats` 拉用量增量
- `monitor_traffic_daily`：每日聚合方便出图
- 前端：会员前台显示用量进度条，运营后台看趋势

#### 4.3 IP 健康检查 (resource_ip_compat) ≈ 1 天
- 探测 IP 对常见目标的可达性：OpenAI / Claude / Cloudflare / Google
- 探测方式：从中转通过 SOCKS5 拉 https://chat.openai.com/cdn-cgi/trace 类似 endpoint
- 失败次数超阈值标 `degraded`，每天定时探活
- 运营后台显示每个 IP 的兼容性矩阵

#### 4.4 反向 Reconciler（孤儿清理）≈ 0.5 天
- 拉 Xray gRPC 当前所有 user，对照 DB，清理"远端有但 DB 无"的 user

### Phase 5 · 商业化优化 ⏳ ≈ 5 天（按优先级取舍）

| 功能 | 工时 | 价值 |
|---|---|---|
| 续费/换 IP 自助 | 1 天 | 高（减少客服） |
| 操作日志查询页 | 0.5 天 | 中 |
| 折扣码 / 满减 | 1 天 | 中 |
| 平台兼容性展示在套餐详情 | 0.5 天 | 高（卖点） |
| 自助换 IP（同套餐） | 1 天 | 中 |
| 数据分析仪表盘（销售/活跃） | 1 天 | 中 |

## 五、产品化建议

### 售卖渠道分层
- **闲鱼**：纯 CDK 卡密发货，价位 99-199 元（住宅 IP 199, ISP 99）
- **淘宝**：同上，加品牌包装
- **官网**（nook-portal 一旦成型）：299 元独享 ISP，附赠平台兼容性测试报告
- **B 端长租**：单独议价，月付/年付，开发票

### 信任建立（卖家最痛）
- **明码标价 + 兼容性报告**：套餐详情列出「ChatGPT 可用 / Claude 可用 / 测速 280Mbps / Scamalytics 评分 12/100」
- **试用 1 小时**：免费体验链接，吸引犹豫客户
- **48 小时无理由退款**：流量未超 1GB 全额退（拉低决策门槛）
- **闲鱼自动回复**：买家拍下后机器人秒发卡密 + 教程链接

### 价格阶梯
| 套餐 | 价位 | 配置 |
|---|---|---|
| 基础月付 | 99 元/月 | 美国 ISP、不限流量、1 IP |
| 高级月付 | 199 元/月 | 美国住宅 ISP、不限流量、1 IP、平台兼容性报告 |
| 工作室季付 | 499 元/3月 | 美国住宅 ISP、不限流量、3 IP、自助换 IP |
| 企业半年付 | 1199 元/6月 | 多区可选、不限流量、5 IP、专属客服 |

### 风控要点
- **多账号同 IP 监测**：同一 IP 在 X 分钟内被 N 个不同会员访问 → 风险账号
- **用量异常告警**：单会员 1 天用量超 100GB → 可能转售/滥用
- **OpenAI 帐号联防**：买家退款后立即换 IP 防黑产撞库（自助换 IP 流程）
- **法律合规**：服务条款明示禁止违法用途，保留断网权利

### 售后规模化
- **客服 SOP** 文档化：常见问题 30 条 + 标准回复
- **自助工单**：会员前台提交问题，分级（紧急/普通），运营仪表盘看
- **状态页**：`status.nook.x.com` 展示各区可用性，主动公示故障

## 六、风险与备选方案

| 风险 | 缓解 |
|---|---|
| Xray gRPC API 不稳/被 Xray 上游改动 | 抽象层 XrayBackend 已隔离, 切回方案 B (3x-ui HTTP) 工作量约 3-5 天 |
| SOCKS5 落地 IP 被平台批量拉黑 | resource_ip_compat 周期探测 + 自助换 IP 流程 |
| 单台中转性能瓶颈 (大量并发用户) | 加中转 server 横向扩；nook 已支持多 server |
| 闲鱼下架 / 改规则 | 多渠道铺货，主推官网 |
| 产品被同质化竞争压价 | 比拼"兼容性透明度 + 售后响应速度" |
| 法律风险 (国内代理服务灰色) | 明确"仅供海外学术研究/跨境电商" + 不主动宣传翻墙 |
