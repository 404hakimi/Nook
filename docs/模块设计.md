## 一、目录结构

```
nook/                                       # 根 pom (packaging=pom)
├── pom.xml
├── nook-framework/                         # 基础设施聚合 pom (RuoYi-Plus 风格)
│   ├── pom.xml
│   ├── nook-common/                        # POJO 基础: Result / ErrorCode / BusinessException / HandlerOrder
│   ├── nook-spring-boot-starter-mybatis/   # ORM: BaseEntity / MetaObjectHandler / MybatisPlusConfig
│   ├── nook-spring-boot-starter-web/       # GlobalExceptionHandler / FastJson2Config / ClientIpResolver
│   ├── nook-spring-boot-starter-redis/     # Redisson 客户端 + RedisTemplate JSON 序列化
│   └── nook-spring-boot-starter-security/  # sa-token + BCryptPasswordEncoder
├── nook-biz-module/                        # 业务聚合 pom (packaging=pom)
│   ├── pom.xml
│   ├── nook-biz-system/                    # 后台用户体系  (system_*)
│   ├── nook-biz-member/                    # 会员体系      (member_*)
│   ├── nook-biz-resource/                  # 服务器/IP     (resource_*)
│   ├── nook-biz-xray/                      # Xray gRPC 对接 (xray_*)
│   ├── nook-biz-business/                  # 套餐/CDK/订单/直充 (business_*)
│   └── nook-biz-monitor/                   # 流量统计 + 健康检查 + 告警 (monitor_*)
├── nook-server/                            # 启动模块: 引入全部 biz，含 application.yml，最终打 jar
│   └── pom.xml
└── nook-admin/                             # 后台前端 (Vue 3 + TS + Vite + DaisyUI)
    └── package.json
```

根 pom 只感知 3 个直接子模块（framework / biz-module / server）。基础设施模块归拢在 `nook-framework/` 下，业务模块归拢在 `nook-biz-module/` 下。两个聚合 pom **不做 dependencyManagement**，所有子模块 `<parent>` 仍指向**根 pom**，避免双层继承。

## 二、模块职责

| 模块 | packaging | 职责 |
|------|-----------|------|
| `nook` | pom | 根聚合，统一 Spring Boot / 第三方依赖版本 |
| `nook-framework` | pom | 基础设施聚合，含 5 个子模块。详见 §三 |
| &nbsp;&nbsp;&nbsp;`nook-common` | jar | POJO 基础: `Result` / `ErrorCode` / `BusinessException` / `HandlerOrder` |
| &nbsp;&nbsp;&nbsp;`nook-spring-boot-starter-mybatis` | jar | ORM: `BaseEntity` / `MetaObjectHandler` / `MybatisPlusConfig` |
| &nbsp;&nbsp;&nbsp;`nook-spring-boot-starter-web` | jar | Web MVC: `GlobalExceptionHandler` / `FastJson2Config` / `ClientIpResolver` |
| &nbsp;&nbsp;&nbsp;`nook-spring-boot-starter-redis` | jar | Redisson + 自定义 RedisTemplate 序列化 |
| &nbsp;&nbsp;&nbsp;`nook-spring-boot-starter-security` | jar | sa-token + BCryptPasswordEncoder |
| `nook-biz-module` | pom | 业务聚合 pom，自身无代码，只 aggregate 6 个 biz 子模块 |
| `nook-biz-system` | jar | 后台用户、登录、操作日志、Controller(`/admin/system/**`) |
| `nook-biz-member` | jar | 会员账户、设备、操作日志、订阅、Controller(`/portal/member/**`) |
| `nook-biz-resource` | jar | 服务器、IP 类型、IP 池、IP 兼容性、Controller |
| `nook-biz-xray` | jar | gRPC 客户端、Inbound 增删改、Reconciler、SSH 隧道（**含 protobuf 编译插件**） |
| `nook-biz-business` | jar | 套餐、CDK、后台直充、订单流水、Controller |
| `nook-biz-monitor` | jar | **观测域**：5 分钟流量快照、日报、服务器带宽、IP 健康检查、平台兼容性探测、告警 |
| `nook-server` | jar | **启动主类 + `application.yml` + Spring Boot 打包**，依赖全部 biz，最终产出 `nook-server.jar` |

每个 biz 模块**自己负责自己的 Controller**，没有独立的 web 聚合层。

## 三、`nook-framework` 内部分层

`nook-framework` 是聚合 pom，按职责拆 5 个子模块（参考 RuoYi-Plus 风格）：

### nook-common — POJO 基础（无 Spring 依赖）

```
com.nook.common.web
├── response/   Result                  统一返回结构
├── error/      ErrorCode/CommonErrorCode  错误码契约 + 通用错误码
├── exception/  BusinessException       业务异常基类
└── constant/   HandlerOrder            @RestControllerAdvice 顺序常量
```

仅依赖 hutool / lombok / jackson-annotations。任何 starter-* 子模块均可引入此模块。

### nook-spring-boot-starter-mybatis — ORM 基础

```
com.nook.framework.mybatis
├── entity/     BaseEntity              id + 审计字段(createdAt/updatedAt) 实体基类
├── handler/    MetaObjectHandlerImpl   时间字段自动填充
└── config/     MybatisPlusConfig       分页、乐观锁拦截器、@MapperScan
```

依赖 nook-common + mybatis-plus-spring-boot3-starter。biz 模块的实体类继承 `BaseEntity`。

### nook-spring-boot-starter-web — Web MVC

```
com.nook.framework.web
├── ClientIpResolver                       从代理头解析真实客户端 IP
├── config/FastJson2Config                 注册 fastjson2 HttpMessageConverter
└── exception/GlobalExceptionHandler       全局异常 → Result
```

依赖 nook-common + spring-boot-starter-web + spring-boot-starter-validation + fastjson2 + knife4j。

### nook-spring-boot-starter-redis — Redis 客户端

```
com.nook.framework.redis
└── config/NookRedisAutoConfiguration      自定义 RedisTemplate(JSON + JavaTime 序列化)
```

依赖 nook-common(hutool 反射工具) + redisson-spring-boot-starter + jackson-datatype-jsr310。
通过 `META-INF/spring/...AutoConfiguration.imports` 注册为 Spring Boot 自动配置；自动读取 `spring.data.redis.*` 创建 `RedissonClient`。

### nook-spring-boot-starter-security — 鉴权

```
com.nook.framework.security
├── stp/StpSystemUtil + StpMemberUtil      sa-token 多账号 (system / member)
├── config/SaTokenConfig                   /admin/** /portal/** 路由拦截
├── config/PasswordEncoderConfig           BCryptPasswordEncoder Bean
└── handler/SaTokenExceptionHandler        NotLoginException → Result 401
```

依赖 starter-web (使用 @RestControllerAdvice) + starter-redis (sa-token-redisson-jackson 复用 RedissonClient)。

### 依赖图

```
                      starter-mybatis ─┐
                                       │
biz 模块  ────→  starter-security ──┬──→ starter-web ──→ nook-common
                                    └──→ starter-redis ──→ nook-common
```

biz 模块显式声明的依赖：
- `nook-spring-boot-starter-security` (传递: web + redis + common)
- `nook-spring-boot-starter-mybatis` (并行依赖, 提供 BaseEntity 与 MybatisPlusConfig)

### 怎么判断新代码该放哪？

| 判断 | 放哪 |
|---|---|
| 是 POJO / 异常 / 错误码？ | nook-common |
| ORM / 实体基类 / MyBatis 拦截器？ | nook-spring-boot-starter-mybatis |
| 跟 Web 请求/响应/异常处理相关？ | nook-spring-boot-starter-web |
| Redis / 缓存 / 分布式锁？ | nook-spring-boot-starter-redis |
| 鉴权 / 加密 / Token？ | nook-spring-boot-starter-security |
| 本项目业务？ | 对应 `nook-biz-*` |

## 四、模块内部包结构

每个 `nook-biz-<sub>` 模块按以下结构组织：

```
src/main/java/com/nook/biz/<sub>/
├── api/                       # 【对外契约】其他 biz 模块只允许引用此包
│   ├── XxxApi.java            #   对外开放的 Service 接口
│   └── dto/
│       └── XxxDTO.java        #   接口入参/出参 DTO
├── controller/                # REST API（admin/portal 通过 URL 前缀区分）
│   ├── admin/
│   │   └── XxxAdminController.java
│   └── portal/
│       └── XxxPortalController.java
├── service/                   # 内部实现
│   ├── XxxService.java        #   模块内部 Service 接口
│   └── impl/
│       ├── XxxServiceImpl.java
│       └── XxxApiImpl.java    #   实现 api/XxxApi
├── mapper/                    # MyBatis Mapper
├── entity/                    # 数据库实体（继承 BaseEntity）
├── dto/                       # 模块内部 DTO（不对外）
├── vo/                        # Controller 出参 VO
├── constant/                  # 错误码、Redis Key、状态枚举
└── exception/                 # 业务异常
```

### 跨模块引用规则

- **其他模块只能 import `com.nook.biz.<sub>.api.*`**，不允许碰 controller / service / mapper / entity / dto / vo。
- `api` 包是该模块对外的"窄接口"，包含两类东西：
  1. `XxxApi` 接口：定义跨模块调用的能力
  2. DTO：接口的入参/出参，**不要**直接返回 entity
- `api` 接口的实现类放在本模块的 `service/impl/XxxApiImpl.java`，加 `@Service` 由 Spring 注入。
- 这是**约定式**边界，编译器不强制；review 时把关。
- 后续若需要拆服务（Dubbo / Feign），把 `api` 包独立成 `nook-biz-<sub>-api` 子模块即可，业务实现侧不受影响。

## 五、模块依赖关系

```
nook-server
  └→ nook-biz-system
  └→ nook-biz-member
  └→ nook-biz-business
  └→ nook-biz-resource
  └→ nook-biz-xray
  └→ nook-biz-monitor
  └→ nook-framework            (传递依赖也会拉到，显式列出更清晰)

biz 模块之间的引用（仅依赖对方的 api 包）：
  nook-biz-business → nook-biz-member, nook-biz-resource, nook-biz-xray
  nook-biz-monitor  → nook-biz-resource, nook-biz-member, nook-biz-xray
  nook-biz-xray     → nook-biz-resource

所有 nook-biz-* → nook-framework → nook-common
```

依赖原则：
- biz 模块之间允许引用，**只引用对方的 `api` 包**
- 单向依赖，禁止循环（如 member → business 是禁止的）
- 没有中间聚合层，没有 web 聚合层

## 六、parent 继承

**所有子模块的 `<parent>` 都直接指向根 pom**（不指向 `nook-biz-module`），避免双层继承。`nook-biz-module` 只是 `<modules>` 聚合器。

| 模块位置 | relativePath |
|---|---|
| `nook-common/pom.xml` | `../pom.xml` |
| `nook-framework/pom.xml` | `../pom.xml` |
| `nook-biz-module/pom.xml` | `../pom.xml` |
| `nook-biz-module/nook-biz-system/pom.xml` 等 6 个 | `../../pom.xml` |
| `nook-server/pom.xml` | `../pom.xml` |

```xml
<!-- nook-biz-module/nook-biz-system/pom.xml 示例 -->
<parent>
    <groupId>com.nook</groupId>
    <artifactId>nook</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <relativePath>../../pom.xml</relativePath>
</parent>
```

根 pom 的 `<dependencyManagement>` 统一管理所有第三方版本和内部模块版本。

## 七、`<modules>` 列表

**根 pom**：

```xml
<modules>
    <module>nook-common</module>
    <module>nook-framework</module>
    <module>nook-biz-module</module>     <!-- 聚合，自动包含 6 个 biz -->
    <module>nook-server</module>
</modules>
```

**`nook-biz-module/pom.xml`**：

```xml
<modules>
    <module>nook-biz-system</module>
    <module>nook-biz-member</module>
    <module>nook-biz-resource</module>
    <module>nook-biz-xray</module>
    <module>nook-biz-business</module>
    <module>nook-biz-monitor</module>
</modules>
```

构建：

```bash
mvn clean package -pl nook-server -am
java -jar nook-server/target/nook-server.jar
```

`-am` 自动按依赖顺序构建上游模块。最终只有 `nook-server` 产出可执行 jar。

## 九、决议变更日志

| 项 | 决议 | 备注 |
|---|---|---|
| 模块前缀 | 用 `nook-biz-*` | RuoYi 风格 |
| 中间聚合 pom (`nook-module`) | ❌ 取消 | 改用 `nook-biz-module` 只聚合 biz 子模块 |
| 业务聚合 pom (`nook-biz-module`) | ✅ 新增 | 仅作 `<modules>` 聚合，不做依赖管理；6 个 biz 物理位置都在它下面 |
| Web 入口模块 (`nook-web`) | ❌ 取消 | Controller 写在各 biz 模块的 `controller/` 下 |
| 启动模块 | ✅ 新增 `nook-server` | 含 `main` + `application.yml` + 打 jar |
| biz 模块切分粒度 | 6 个不再细拆 | system / member / business / resource / xray / monitor (含统计) |
| statistics + monitor 合并 | ✅ 合为 `nook-biz-monitor`，含流量统计 + 健康检查 + 告警 | — |
| biz 之间引用 | ✅ 允许，仅限对方的 `api` 包 | — |
| SQL 拆分 | ✅ 按模块 7 文件 + 种子 | 见 [sql/README.md](../sql/README.md) |
| 索引/分区/外键 | 现阶段全部不加 | 见 [docs/后端开发规范.md §2.4](后端开发规范.md) |
| common vs framework 边界 | ✅ 明确：common = 跨项目轻基础，framework = 项目特异性配置 | 见 §三 |
| common 子包再分层 | ✅ `mybatis/{entity,handler,config}` + `web/{response,error,exception}` | 不平铺 |
| 鉴权方案 | ✅ sa-token 1.39.0 + Redis-Jackson 持久化 | 多账号体系 `system` / `member` |
| 后台前端 | ✅ `nook-admin/`（Vue 3 + TS + Vite + Element Plus） | PC 客户端暂不做 |
