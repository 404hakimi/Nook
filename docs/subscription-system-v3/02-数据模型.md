## 1. 通用约定

### 1.1 主键

业务表用 `id CHAR(32)` UUID, `@TableId(type = IdType.ASSIGN_UUID)`, 由 MyBatis-Plus 在 INSERT 时生成 32 位无连字符 hex.

1:1 子表用上级表的 FK 作 PK (e.g., `resource_server_credential.server_id` PK = `resource_server.id`).

字典表用业务码作 PK (e.g., `resource_region.code = 'JP-TYO'`).

### 1.2 时间字段

主表 (含 `BaseEntity`): `created_at` / `updated_at`, 由 `MetaObjectHandlerImpl` 在 INSERT / UPDATE 时自动 fill.

1:1 子表不继承 `BaseEntity` (PK 不是 id), 字段手动 set 或在 Mapper `update` 时显式 `set updated_at = NOW()`.

### 1.3 软删除

`@TableLogic Integer deleted` 仅用于:
- `resource_server.deleted`
- `resource_ip_pool.deleted`
- `member_user.deleted`

1:1 子表不软删: 通过 join 主表 `deleted = 0` 过滤即可, 避免双层冗余.

### 1.4 Update 策略

MP 全局 `mybatis-plus.global-config.db-config.update-strategy: NOT_NULL`. 所有 `update(entity, wrapper)` 跳过 null 字段 — **PATCH 语义** (不能通过 update 把字段从有值清成 null). 这是有意决策, 见 [07-决策](./07-决策-排除-风险-Checklist.md#patch-语义).

### 1.5 跨模块 API

业务模块拆 `nook-module-<x>-api` (对外契约: enums + DTO + Api interface) + `nook-module-<x>-server` (实现). 跨模块调用只走 `-api` 包, 不直接注入对方 Service / Mapper.

### 2.1 会员

#### `member_user` — 会员账户 ✅

```sql
CREATE TABLE member_user (
  id              CHAR(32)     PRIMARY KEY,
  email           VARCHAR(128) UNIQUE NOT NULL,
  password_hash   VARCHAR(255) NOT NULL              COMMENT 'BCrypt 哈希',
  sub_token       CHAR(32)     UNIQUE NOT NULL      COMMENT '用户级聚合订阅 URL token, 32 char hex',

  last_login_at   DATETIME,

  remark          VARCHAR(255),                      -- admin 备注
  created_at      DATETIME     NOT NULL,
  updated_at      DATETIME     NOT NULL,

);
```

`sub_token` 永久有效 (除非用户主动重置), portal 端拼 `https://nook.com/sub/{sub_token}` 返回该用户所有活跃 sub 的 vmess 节点. 详见 [03-业务核心 §7](./03-业务核心-算法与流程.md).

### 2.2 资源 - server 系列

`resource_server` 主表 + 5 张子表, 按更新频率分层 (见 [§4.2 三表分层原理](#42-三表分层)).

#### `resource_server` — 主表 (低频, 装机时入) ✅

```sql
CREATE TABLE resource_server (
  id                CHAR(32)     PRIMARY KEY,
  name              VARCHAR(64)  NOT NULL                            COMMENT '别名 (admin UI 显示)',
  lifecycle_state   ENUM('INSTALLING','READY','LIVE','RETIRED') NOT NULL DEFAULT 'INSTALLING'
                                                                     COMMENT '装机生命周期; 走 transition 接口流转',
  total_ip_count    INT          NOT NULL DEFAULT 0                  COMMENT 'IP 总数 (展示用, 现状无业务自动写入)',
  region            VARCHAR(64),                                     -- FK → resource_region.code
  remark            VARCHAR(512),
  agent_token       CHAR(64)     UNIQUE                              COMMENT 'Agent 鉴权 token; 装机一次性 SHA256 生成, 永不更新',
  created_at        DATETIME     NOT NULL,
  updated_at        DATETIME     NOT NULL,
  deleted           TINYINT      NOT NULL DEFAULT 0,
  INDEX idx_lifecycle (lifecycle_state, deleted),
  CONSTRAINT fk_server_region FOREIGN KEY (region) REFERENCES resource_region(code)
);
```

**Lifecycle 状态机** (ResourceServerLifecycleEnum):
```
INSTALLING ↔ READY ↔ LIVE → RETIRED → LIVE
                     ↑
                     LIVE 前置: resource_server_dns.domain 必填
```

允许的转移 (`ResourceServerServiceImpl.ALLOWED_LIFECYCLE_TRANSITIONS`):
- INSTALLING ↔ READY
- READY ↔ LIVE
- READY → RETIRED / LIVE → RETIRED
- RETIRED → LIVE (复活)

**仅 LIVE** 参与 allocator 候选; RETIRED 老客户走退役切换 (见 [03 §10](./03-业务核心-算法与流程.md)).

#### `resource_server_credential` — SSH 凭据 (1:1) ✅

```sql
CREATE TABLE resource_server_credential (
  server_id                  CHAR(32)     PRIMARY KEY,
  host                       VARCHAR(128) NOT NULL                  COMMENT 'SSH IP / 域名',
  ssh_port                   INT          NOT NULL DEFAULT 22,
  ssh_user                   VARCHAR(64),                            -- 默认 root
  ssh_password               VARCHAR(255),                           -- 明文 (后台受信网络); update 留空 = 保留原值
  ssh_timeout_seconds        INT,                                    -- 握手超时
  ssh_op_timeout_seconds     INT,                                    -- 单命令超时
  ssh_upload_timeout_seconds INT,                                    -- SCP 超时
  install_timeout_seconds    INT,                                    -- 装机整体超时
  created_at                 DATETIME     NOT NULL,
  updated_at                 DATETIME     NOT NULL,
  CONSTRAINT fk_credential_server FOREIGN KEY (server_id) REFERENCES resource_server(id)
);
```

**字段约束**:
- LIVE 后 `host` / `ssh_port` **硬锁** (`ResourceServerCredentialValidator.validateUpdate`) — 防止断 agent 心跳
- 密码留空保留原值 (`ResourceServerCredentialServiceImpl.update` 显式 `setSshPassword(null)`)

#### `resource_server_billing` — 账面 (1:1) ✅

```sql
CREATE TABLE resource_server_billing (
```

全套字段都是 admin 录入, 后端无业务读取参与 allocator 决策. v3 不主动限速, `bandwidth_mbps` 仅商品页展示用.

#### `resource_server_dns` — DNS 绑定 (1:1) ✅

```sql
CREATE TABLE resource_server_dns (

  cf_zone_id    VARCHAR(64)                                         COMMENT 'Cloudflare Zone ID',
```

#### `resource_server_capacity` — NIC 流量监控 (1:1, 中频 5min) 🟡

```sql
CREATE TABLE resource_server_capacity (
  server_id            CHAR(32)     PRIMARY KEY,
  monthly_traffic_gb   INT                                          COMMENT '机房 NIC 月配额 GB (0/null=不限)  [TODO 无写入入口]',

                                                                    COMMENT '周期重置策略',

  updated_at           DATETIME     NOT NULL,
```

**已实现**:

**未实现 (TODO)**:
- `monthly_traffic_gb` 无 admin UI 编辑入口

#### `resource_server_runtime` — 心跳运行时 (1:1, 高频 1min) ✅

```sql
CREATE TABLE resource_server_runtime (
  server_id            CHAR(32)     PRIMARY KEY,
  last_heartbeat_at    DATETIME                                     COMMENT 'agent 上次心跳; null=从未',
  temp_unhealthy       TINYINT      NOT NULL DEFAULT 0              COMMENT '心跳 3-5min 缺失暂时不健康',
  agent_version        VARCHAR(32)                                  COMMENT 'agent 自报版本',
  last_agent_seen_ip   VARCHAR(64)                                  COMMENT 'agent 源 IP (反查网络)',
  consecutive_miss     INT          NOT NULL DEFAULT 0              COMMENT '连续缺失次数; 心跳到达清零',
  updated_at           DATETIME     NOT NULL,                       -- 无 created_at, init 时即占位
  CONSTRAINT fk_runtime_server FOREIGN KEY (server_id) REFERENCES resource_server(id)
);
```

**派生在线状态** (前端展示, 不入库): 见 `AgentOnlineState.classify`:
- elapsed < 60s → ONLINE
- 60-180s → WARN
- 180-300s 或 temp_unhealthy=1 → TEMP_UNHEALTHY
- ≥ 300s → OFFLINE
- last_heartbeat_at=null → NEVER (未装/装中)

后端 `AgentHeartbeatTimeoutJob` 周期扫表把 `consecutive_miss` 累加, 满 3 次置 `temp_unhealthy=1`.

### 2.3 资源 - IP 系列

#### `resource_ip_pool` — IP 池 (落地 SOCKS5) ✅

字段较多, 按语义分组:

```sql
CREATE TABLE resource_ip_pool (
  -- 身份 / 路由
  id                    CHAR(32)     PRIMARY KEY,
  ip_address            VARCHAR(45)  UNIQUE NOT NULL                COMMENT '出网真实 IP, 用户对外暴露',
  region                VARCHAR(64)  NOT NULL                       COMMENT 'FK → resource_region.code',
  ip_type_id            CHAR(32)     NOT NULL                       COMMENT 'FK → resource_ip_type.id (ISP/家宽/机房)',

  -- 生命周期 + 占用状态 (双层状态机, 见 §4.1)

                                                                    COMMENT '装机阶段',
  status                ENUM('AVAILABLE','RESERVED','OCCUPIED','COOLING') NOT NULL DEFAULT 'AVAILABLE'
                                                                    COMMENT 'allocator 占用状态',
  provision_mode        TINYINT      NOT NULL DEFAULT 2             COMMENT '1=self_deploy(SSH+SOCKS5一键部署) 2=external(仅记录第三方现成)',

  -- 占用追踪
  occupied_by_member_id CHAR(32)                                    COMMENT 'status=OCCUPIED 时填占用会员',
  occupied_at           DATETIME,
  cooling_until         DATETIME                                    COMMENT 'status=COOLING 时填; 退订后冷却避免历史污染',
  reserved_expires_at   DATETIME                                    COMMENT 'status=RESERVED 时填; 30min 预占',
  assign_count          INT          NOT NULL DEFAULT 0             COMMENT '历史分配次数',
  last_health_at        DATETIME,

  -- SOCKS5 服务参数
  socks5_port           INT          NOT NULL,
  socks5_username       VARCHAR(64)                                  COMMENT '明文存储 (待加密)',
  socks5_password       VARCHAR(255)                                COMMENT '明文存储 (待加密)',
  log_level             VARCHAR(64)  NOT NULL DEFAULT 'connect disconnect error'
                                                                    COMMENT 'dante log 关键字组合 (空格分隔)',
  log_path              VARCHAR(255)                                COMMENT 'dante logoutput; NULL = install_dir/logs/sockd.log',
  autostart_enabled     TINYINT      NOT NULL DEFAULT 1             COMMENT 'systemd 开机自启',
  firewall_enabled      TINYINT      NOT NULL DEFAULT 1             COMMENT '部署时配 UFW',
  firewall_allow_from   VARCHAR(255)                                COMMENT 'UFW allow CIDR; NULL=0.0.0.0/0',
  install_dir           VARCHAR(255) NOT NULL DEFAULT '/home/socks5' COMMENT 'dante 安装目录; danted.conf 仍在 /etc/danted.conf',

  -- SSH 部署 (provision_mode=1 才用)
  ssh_host              VARCHAR(128)                                 COMMENT '留空 = ip_address 兜底',
  ssh_port              INT                                          COMMENT '留空 = 22',
  ssh_user              VARCHAR(64)                                  COMMENT '留空 = root',
  ssh_password          VARCHAR(255),

  -- 限速 + 账面
  bandwidth_limit_mbps  INT          NOT NULL DEFAULT 0             COMMENT '落地机 dante 实际限速 Mbps; 0=不限; agent 改 sockd.conf 落实, 见 §5.2  [TODO 字段未加]',
  bandwidth_mbps        INT                                          COMMENT '采购带宽上限 (Mbps); 仅账面, 不参与 enforce',
  traffic_quota_gb      INT                                          COMMENT '采购流量上限 (GB); 仅账面',
  cost_monthly_usd      DECIMAL(10,2),
  billing_cycle_day     TINYINT                                      COMMENT '账单日 1-28',
  expires_at            DATE,
  remark                VARCHAR(255),

  -- Agent 鉴权 (落地机也跑 nook-agent, 见 §5.2 / 05-Agent架构)
  agent_token           CHAR(64)     UNIQUE                          COMMENT 'Agent 鉴权 token; 装机一次性 SHA256 生成  [TODO 字段未加]',

  created_at            DATETIME     NOT NULL,
  updated_at            DATETIME     NOT NULL,
  deleted               TINYINT      NOT NULL DEFAULT 0,

  INDEX idx_pool_filter (lifecycle_state, status, ip_type_id, region) COMMENT 'allocator 候选过滤',
  CONSTRAINT fk_ip_region FOREIGN KEY (region) REFERENCES resource_region(code)
);
```

**双层状态机**:
- `lifecycle_state` 装机阶段 (admin 视角): `INSTALLING → READY → LIVE → RETIRED`, 跟 server 完全一致流转
- `status` 占用状态 (allocator 视角, 仅 lifecycle=LIVE 期间有意义):
  ```
  AVAILABLE → RESERVED (30min 支付预占)
            → OCCUPIED (订阅期间)
            → COOLING  (退订后 ip_type.cooling_minutes 内禁止复用)
            → AVAILABLE
  ```

#### `xray_node` — Xray 实例配置 (1:1 PK=server_id) ✅

```sql
CREATE TABLE xray_node (
  server_id            CHAR(32)     PRIMARY KEY,                    -- FK → resource_server.id
  xray_version         VARCHAR(32)  NOT NULL DEFAULT 'v1.8.23',
  xray_api_port        INT          NOT NULL DEFAULT 8080           COMMENT 'API server loopback 端口',

  -- 共享 inbound 配置 (当前部署期固定 vmess + ws + 0.0.0.0)
  protocol             VARCHAR(16)  NOT NULL DEFAULT 'vmess'        COMMENT '当前固定; 协议适配阶段放开',
  transport            VARCHAR(32)  NOT NULL DEFAULT 'ws',
  listen_ip            VARCHAR(45)  NOT NULL DEFAULT '0.0.0.0',
  shared_inbound_port  INT                                          COMMENT '共享端口 (默认 443)',
  ws_path              VARCHAR(128)                                 COMMENT 'WebSocket path (CDN 接入)',
  domain               VARCHAR(255)                                 COMMENT '节点对外域名 (CDN CNAME)',
  tls_cert_path        VARCHAR(255)                                 COMMENT 'acme.sh 签发后填',
  tls_key_path         VARCHAR(255),

  -- 远端路径布局
  xray_install_dir     VARCHAR(255),
  xray_log_dir         VARCHAR(255) NOT NULL DEFAULT '/var/log/xray',
  xray_binary_path     VARCHAR(255),                                -- = $INSTALL_DIR/bin/xray
  xray_config_path     VARCHAR(255),                                -- = $INSTALL_DIR/etc/xray/config.json
  xray_share_dir       VARCHAR(255),                                -- geo*.dat

  touchdown_size       INT                                          COMMENT '该 server 最多挂载客户端数 (= 软上限); allocator 强校验',
  last_xray_uptime     DATETIME                                     COMMENT '上次探测的 xray 启动时间 (replay 判定)',
  installed_at         DATETIME                                     COMMENT '最近部署时间; 重装会覆写',
  created_at           DATETIME     NOT NULL,
  updated_at           DATETIME     NOT NULL

```

**关键字段**:
- `touchdown_size` — **核心容量约束**. allocator 候选过滤里 `< COUNT(active clients)` 才允许新开通. xray reinstall 时 `XrayNodeValidator.validateAgainstActiveClients` 校验不能缩到比当前活客户数更小.
- v3 通过 touchdown_size 间接管控带宽分摊 (见 §5 带宽策略).

#### `xray_client` — Xray 客户端 (1 client = 1 user 1 IP) ✅

```sql
CREATE TABLE xray_client (
  id              CHAR(32)     PRIMARY KEY,
  server_id       CHAR(32)     NOT NULL                              COMMENT '线路机 ID',
  ip_id           CHAR(32)     UNIQUE NOT NULL                       COMMENT 'FK → resource_ip_pool.id; UNIQUE: 1 IP 只能挂 1 客户',
  member_user_id  CHAR(32)     NOT NULL,
  client_uuid     VARCHAR(64)  NOT NULL                              COMMENT 'vless/vmess UUID / trojan 密码',
  client_email    VARCHAR(128) NOT NULL                              COMMENT 'member_{会员ID}_{IP ID}',
  status          TINYINT      NOT NULL DEFAULT 1                    COMMENT '1=运行 2=已停 3=待同步 4=远端已不存在  [TODO 待补 enum 类]',
  last_synced_at  DATETIME                                            COMMENT '与远端 xray 对账时间',
  created_at      DATETIME     NOT NULL,
  updated_at      DATETIME     NOT NULL
);
```

**核心约束**: `ip_id UNIQUE` 实现 v3 "1 IP = 1 客户" VPS 实例模式.

**操作流程**:
- `provisionXrayClient` (走 op orchestrator): IP 池 acquire → insert client → SSH 远端 `xray api adu` 加 user

## 4. 设计原则

### 4.1 双层状态机 (lifecycle vs 业务状态)

`resource_server` / `resource_ip_pool` / `sub_main` 都把状态拆两层:

| 表 | 维度 A (低频, admin 视角) | 维度 B (高频, allocator/业务视角) |
|---|---|---|
| `resource_server` | `lifecycle_state` (装机) | `resource_server_capacity.throttle_state` (容量) |
| `resource_ip_pool` | `lifecycle_state` (装机) | `status` (占用) |
| `sub_main` | `status` (业务: ACTIVE/EXPIRED/CANCELLED) | `provision_state` (流程: PROVISIONED/PROVISIONING/ERROR) |

**好处**:
- admin "下架某 server" (lifecycle → RETIRED) 不影响在跑客户; allocator 自然停止派单
- "用户付了费但远端 xray 同步失败" 可表达 (status=ACTIVE + provision_state=ERROR, admin 重试中)

**通用规则**: 仅当 lifecycle_state=LIVE / 业务 status=ACTIVE 时, 高频维度才被业务读取.

### 4.2 三表分层 (按更新频率)

`resource_server` 拆成 主表 + capacity + runtime, 因为更新频率差 100 倍以上:

| 表 | 更新频率 | 字段例 |
|---|---|---|
| `resource_server` (低频) | 装机一次 + 偶尔改 | host, agent_token, region, lifecycle_state |
| `resource_server_capacity` (中频 5min) | NIC 上报 | rx/tx/used_traffic_bytes |
| `resource_server_runtime` (高频 1min) | 心跳 | last_heartbeat_at, consecutive_miss |

**为什么必须拆**:
- 心跳每 1min UPDATE, 一台机器一年 50w+ 次
- 心跳写主表会让主表 buffer pool 命中率下降, SELECT (admin 列表) 受影响
- 拆出独立 runtime: 主表"冷数据" 不被高频写锁污染

Sub-server-credential / billing / dns 也按"更新频率 / 业务职责" 拆出来, 但更多是**职责单一**考虑 (SSH 凭据跟域名跟账面无关, 不应挤在一张表).

### 4.3 表前缀

| 模块 | 前缀 | 表 |
|---|---|---|
| 资源 | `resource_*` | resource_region / resource_server* / resource_ip_pool / resource_ip_type |
| 会员 | `member_*` | member_user |
| 商品 | `plan_*` | plan_sku, plan_sku_resource |
| 订阅 | `sub_*` | sub_main |
| 订单 | `order_*` | order_main, order_item |
| 告警 | `alert_*` | alert_record |
| Agent | `agent_*` | agent_runtime_config, agent_task |
| Xray | `xray_*` | xray_node, xray_client, xray_client_traffic |

## 5. 带宽策略 (落地机 dante + 线路机 tc 双层限速)

### 5.1 链路模型 (容量守恒)

```
用户 ─vmess+ws+TLS─→ [线路机 xray] ─socks5─→ [落地机 dante] ─→ Internet
                          ↓                       ↓
                      tc 出站限速             dante bandwidth class
                      (服务器级)              (per-IP = per-客户, v3 1:1)
```

**核心约束**: 同一线路机上挂载的所有落地机 dante 限速之和, 不能超过线路机 tc 限速 × (1 - 预留率).

**预留率默认 10%** (常量 `BANDWIDTH_RESERVATION_RATIO = 0.10`):

```
Σ(挂在 server X 的所有 IP.bandwidth_limit_mbps) ≤ server X.bandwidth_limit_mbps × 0.9
```

例: 线路机 500 Mbps → 可用 450 Mbps; 挂 9 个 50 Mbps 落地机 ✅ (共 450); 挂 10 个 50 Mbps ❌ (共 500 > 450).

### 5.2 限速实施层

| 层 | 实施方 | 颗粒度 | 配置方式 |
|---|---|---|---|
| **落地机** | 落地机 agent → dante | per-IP (= per-客户, v3 1:1 关系) | `sockd.conf` 加 `bandwidth class` block + `systemctl restart danted` |
| **线路机** | 线路机 agent → Linux tc | 整个出站接口 (服务器级总带宽兜底) | `tc qdisc replace dev eth0 root tbf rate Xmbit burst 100mb latency 50ms` |

**为什么不用 xray 客户级限速**: xray-core 原生不支持 per-user bandwidth throttle, 第三方插件不稳定.

**为什么落地机是 per-IP 颗粒**: v3 模式 1 IP = 1 客户 (`xray_client.ip_id UNIQUE`), 所以 IP 级限速天然等价客户级.

### 5.3 双层容量校验

| 时机 | 类型 | 校验对象 | 公式 | 失败后果 |
|---|---|---|---|---|
| admin 改 IP / 线路机限速 + 加 IP 到 SKU 池 | **静态** | 资源池整体配置一致性 | `Σ(SKU 池里所有 IP.bandwidth_limit_mbps) ≤ min(SKU 池里所有 server.bandwidth_limit_mbps) × 0.9` | admin 拒绝操作 + 弹窗给具体超载数值 |
| allocator 给新订阅选 server 时 | **动态** | 选这台 server 给这个新 IP 是否安全 | `该 server 当前已挂 IP 的 Σ(bandwidth_limit_mbps) + 新 IP.bandwidth_limit_mbps ≤ server.bandwidth_limit_mbps × 0.9` | allocator 跳过该 server, 选下一个 |

> 静态校验**松约束** (容量平均 OK), 动态校验**紧约束** (实时不超载). 两层都过才允许新订阅落地.

> 落地机 1 IP 1 客户的设计意味着 IP 在哪台 server 上挂取决于 allocator 那一刻的选择, 不是固定绑定. 最坏情况 (所有 IP 全挂一台 server) 受动态约束阻止.

### 5.4 限速字段在哪

| 表 | 字段 | 含义 |
|---|---|---|
| `resource_ip_pool.bandwidth_limit_mbps` | INT | 落地机实际限速 (Mbps); 0 = 不限速 |
| `resource_ip_pool.bandwidth_mbps` | INT | 落地机采购账面 (Mbps); 仅展示, 不参与计算 |
| `resource_server_capacity.bandwidth_limit_mbps` | INT | 线路机实际限速 (Mbps); 0 = 不限速 |
| `resource_server_billing.bandwidth_mbps` | INT | 线路机采购账面 (Mbps); 仅展示 |

**账面 vs 实际限速**:
- `bandwidth_mbps` (账面): admin 进货合同上写的, 不一定真实给到
- `bandwidth_limit_mbps` (实际): agent 真实 enforce 的值, 通常 ≤ 账面

### 5.5 限速变更链路 (agent task 派发)

```
admin 改 ip-pool.bandwidth_limit_mbps = 50
  ↓ PUT /admin/resource/ip-pool/{id}/bandwidth
Backend ResourceIpPoolBandwidthValidator
  - 静态校验 (§5.3 静态)
  - UPDATE resource_ip_pool
  - 派 task: INSERT agent_task (host_type=IP_POOL, host_id=ipId, type=socks5_set_bandwidth, payload={mbps:50})
  ↓ 落地机 agent 轮询拉到 (≤ 60s)
Sock5SetBandwidthExecutor (Go agent)
  - 改 /etc/danted.conf 的 bandwidth class
  - systemctl restart danted
  - 上报 task SUCCESS
```

线路机限速变更走同样链路, task 类型为 `server_set_bandwidth`, executor 跑 `tc qdisc replace`.

详见 [05-Agent架构 §6](./05-Agent架构与参数化.md) 任务类型表.

### 5.6 关于 touchdown_size

`xray_node.touchdown_size` 保留作为**辅助硬上限** (单 server 客户数), 防止 CPU/内存层面被压垮 (跟带宽是两回事). 限速由 §5.1 链路模型保证, 客户数密度由 touchdown_size 保证, 两者**独立约束**:

- 限速校验失败 → 不让加 IP 到 SKU 池
- touchdown_size 校验失败 → allocator 选其他 server / 拒绝新购
