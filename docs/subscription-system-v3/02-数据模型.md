## 3.1 新增表

### `resource_region` — 区域字典

```sql
CREATE TABLE resource_region (
  code         VARCHAR(32)  PRIMARY KEY,           -- 'JP-TYO', 'US-LAX', 'DE-FRA', 'HK'
  country_code CHAR(2)      NOT NULL,
  country_name VARCHAR(64)  NOT NULL,
  city         VARCHAR(64),
  display_name VARCHAR(128) NOT NULL,              -- '日本 · 东京'
  flag_emoji   VARCHAR(8),
  enabled      TINYINT      DEFAULT 1,
  created_at   DATETIME     NOT NULL,
  updated_at   DATETIME     NOT NULL
);
```

### `member_user` — 会员 (含用户级订阅 token)

```sql
CREATE TABLE member_user (
  id            CHAR(32)     PRIMARY KEY,
  email         VARCHAR(128) UNIQUE NOT NULL,
  password_hash VARCHAR(128) NOT NULL,
  sub_token     CHAR(32)     UNIQUE NOT NULL,    -- 用户级聚合订阅 URL token (32 char random hex)
  created_at    DATETIME     NOT NULL,
  updated_at    DATETIME     NOT NULL
);
```

> `sub_token` 永久有效 (除非用户主动重置), 自动反映该用户名下所有活跃 subscription. 详见 [03-业务核心-算法与流程.md](./03-业务核心-算法与流程.md) §7.

### `plan_sku` — 套餐 (商品定义)

```sql
CREATE TABLE plan_sku (
  id               CHAR(32)     PRIMARY KEY,
  code             VARCHAR(64)  UNIQUE NOT NULL,    -- 'jp_tyo_residential_100gb_monthly'
  name             VARCHAR(128) NOT NULL,           -- '日本东京 住宅 IP 100GB/月'
  region_code      VARCHAR(32)  NOT NULL,           -- FK → resource_region.code (展示分类用)
  ip_type          ENUM('RESIDENTIAL','ISP','DATACENTER') NOT NULL,
  traffic_gb       INT          NOT NULL,           -- 月配额, 直接写入 xray totalBytes
  bandwidth_mbps   INT          COMMENT '账面带宽 (商品页展示用); v3 不主动限速, 用户能跑满 IP 物理上限',
  period_days      INT          NOT NULL DEFAULT 30,
  price_cny        DECIMAL(10,2) NOT NULL,
  cost_basis_cny   DECIMAL(10,2),                    -- 内部成本核算
  enabled          TINYINT      NOT NULL DEFAULT 1,
  created_at       DATETIME     NOT NULL,
  updated_at       DATETIME     NOT NULL,
  CONSTRAINT fk_plan_sku_region FOREIGN KEY (region_code) REFERENCES resource_region(code),
  INDEX idx_filter (region_code, ip_type, enabled)
);
```

> `region_code` / `ip_type` 字段在 plan_sku 上保留, 仅用于商品分类展示, 不参与 allocator 候选过滤 (候选由下方 `plan_sku_resource` 关联表决定).

### `plan_sku_resource` — SKU ↔ 资源池关联 (v3 核心新表)

```sql
CREATE TABLE plan_sku_resource (
  id              CHAR(32)     PRIMARY KEY,
  plan_sku_id     CHAR(32)     NOT NULL,            -- FK → plan_sku.id
  resource_type   ENUM('SERVER','IP_POOL') NOT NULL,
  resource_id     CHAR(32)     NOT NULL,            -- resource_server.id 或 resource_ip_pool.id
  enabled         TINYINT      NOT NULL DEFAULT 1,  -- admin 临时禁用某资源 (老 sub 保留, 不接新)
  created_at      DATETIME     NOT NULL,
  UNIQUE KEY uk_sku_res (plan_sku_id, resource_type, resource_id),
  INDEX idx_sku (plan_sku_id, resource_type, enabled)
);
```

**语义**:
- 每个 SKU 关联**多个 server** (门禁池, 至少 1, 建议 ≥ 2 以便池内切换)
- 每个 SKU 关联**多个 IP** (公寓池, 决定 SKU 容量)
- Server 可以**跨 SKU 共享** (一台 server 同时为多个 SKU 服务)
- IP 池建议 **SKU 独占**; 技术上允许重叠, 但 admin 要理解"真实容量受所有共享 SKU 的 IP 总数约束"

**容量计算 (核心简化, 砍 `sold_traffic_gb`)**:

```sql
-- SKU 剩余 = 关联 IP 池里 LIVE + AVAILABLE 的数量
SELECT COUNT(*) AS remaining
FROM plan_sku_resource psr
JOIN resource_ip_pool ip ON ip.id = psr.resource_id
WHERE psr.plan_sku_id = ?
  AND psr.resource_type = 'IP_POOL'
  AND psr.enabled = 1
  AND ip.lifecycle_state = 'LIVE'
  AND ip.status = 'AVAILABLE';
```

容量 = LIVE IP 总数; 已售 = OCCUPIED 数; 剩余 = AVAILABLE 数. 一句 SQL.

**SKU 示例**:

| code | region | ip_type | traffic_gb | period_days | price_cny |
|---|---|---|---|---|---|
| jp_tyo_residential_100gb_m | JP-TYO | RESIDENTIAL | 100 | 30 | 80 |
| jp_tyo_residential_500gb_m | JP-TYO | RESIDENTIAL | 500 | 30 | 320 |
| jp_tyo_datacenter_200gb_m | JP-TYO | DATACENTER | 200 | 30 | 60 |
| us_lax_residential_100gb_m | US-LAX | RESIDENTIAL | 100 | 30 | 120 |
| hk_residential_200gb_y    | HK | RESIDENTIAL | 200 | 365 | 800 |

### SKU 字段修改规则 (核心约定)

| 字段 | 可改 | 说明 |
|---|---|---|
| `name`, `bandwidth_mbps`, `cost_basis_cny`, `enabled` | ✅ | 文案/账面/上下架 |
| `traffic_gb`, `period_days`, `region_code`, `ip_type`, `price_cny` | ❌ | **业务核心**, 改了会让历史订阅引用错位 |

**想调价 / 改流量挡 / 换地区 / 换 IP 类型 → 一律建新 SKU, 老 SKU `enabled=0`** 下架.

**好处**:
- `sub_main` 完全**不需要快照字段** — 关键属性永远从 plan_sku 实时读, 因为这些字段永不变
- 用户续费看到旧 SKU 已下架 → portal 提示"该套餐已停售, 请选择新套餐" → 用户主动选新 SKU = 隐式同意新价
- 历史订单流水自然记录于 `order_item.unit_price_cny`, 不丢

> **没有** `plan_tier` / `traffic_tier` / `ip_price` / `bundle_discount`. 价格直接写在 plan 上.
>
> **没有** `ip_count` 字段. 每个 plan = 1 IP. 用户想要 3 IP 就买 3 个套餐.

### `sub_main` — 订阅 (1 sub = 1 plan_sku = 1 IP = 1 server = 1 xray_client)

```sql
CREATE TABLE sub_main (
  id              CHAR(32)     PRIMARY KEY,
  member_user_id  CHAR(32)     NOT NULL,
  plan_id         CHAR(32)     NOT NULL,             -- FK → plan_sku.id
  xray_client_id  CHAR(32)     UNIQUE NOT NULL,      -- FK 1:1 → xray_client.id
  sub_domain      VARCHAR(128) UNIQUE NOT NULL,      -- 该 sub 独享子域名, e.g., jp-alice-001.nook.com
  cf_record_id    VARCHAR(64),                       -- Cloudflare DNS record ID (创建时回填, 切换时用)
  started_at      DATETIME     NOT NULL,
  expires_at      DATETIME     NOT NULL,
  status          TINYINT      NOT NULL,             -- 1=ACTIVE 2=EXPIRED 3=CANCELLED 4=PROVISIONING 5=ERROR
  created_at      DATETIME     NOT NULL,
  updated_at      DATETIME     NOT NULL,
  INDEX idx_member_active (member_user_id, status),
  INDEX idx_expires (expires_at, status),
  INDEX idx_sub_domain (sub_domain)
);
```

> **1:1 关系**: 一个 sub 对应一个 server 上的一个 xray_client. 切换服务器时 UPDATE xray_client.server_id 同时调 Cloudflare API 改 sub_domain DNS 指向. 不预部署多份, 没有 mesh.

> **sub_domain 独享设计**: 每个 sub 一个独立子域名 (生成时 e.g., `${region_lower}-${sub_id_short}.nook.com`). 故障 / 池内切换 / IP 换 IP 时, 用户 vmess URL 永远不变 — 只是 DNS 解析指向变. 用户客户端 5-15 min DNS 缓存过期后自动恢复, 不需要刷订阅.

> **不留快照字段**: 因为 plan_sku 关键字段不可修改, JOIN 实时读永远是当年那个值, 等价于快照.

### `order_main` + `order_item` — 订单 (购物车一次支付, N 个 item)

```sql
CREATE TABLE order_main (
  id                   CHAR(32)     PRIMARY KEY,
  member_user_id       CHAR(32)     NOT NULL,
  amount_cny           DECIMAL(10,2) NOT NULL,
  status               ENUM('PENDING','PAID','REFUNDED','CANCELLED','EXPIRED') NOT NULL,
                                                         -- EXPIRED = 30min 未付款, 自动释放库存
  reserve_expires_at   DATETIME     NOT NULL,            -- 库存预占截止时间 (创建时 = now + 30min)
  paid_at              DATETIME,
  created_at           DATETIME     NOT NULL,
  updated_at           DATETIME     NOT NULL,
  INDEX idx_member_status (member_user_id, status),
  INDEX idx_reserve_expires (reserve_expires_at, status)
);

CREATE TABLE order_item (
  id                  CHAR(32)     PRIMARY KEY,
  order_id            CHAR(32)     NOT NULL,             -- FK → order_main.id
  type                ENUM('NEW','RENEW') NOT NULL,      -- 起步无 ADDON (加购包后续阶段)
  plan_id             CHAR(32),                          -- type=NEW 时填; FK → plan_sku.id
  subscription_id     CHAR(32),                          -- type=RENEW 时填; FK → sub_main.id
  quantity            INT          NOT NULL DEFAULT 1,
  unit_price_cny      DECIMAL(10,2) NOT NULL,
  amount_cny          DECIMAL(10,2) NOT NULL,
  reserved_server_id  CHAR(32),                          -- 创建 order 时锁定的候选 server
  reserved_ip_id      CHAR(32),                          -- 创建 order 时锁定的候选 IP
  created_at          DATETIME     NOT NULL,
  INDEX idx_order (order_id)
);
```

> **同一 NEW item quantity > 1** 时, 拆成 quantity 条独立 item, 每条锁一个 (server, ip). 简化模型.

> **流量包加购功能**: Sprint 1 起步**不做**. 等运营数据足够后再加 `plan_traffic_addon` + `sub_addon` 两张表 (Sprint 5+).

### `alert_record` — 系统告警

```sql
CREATE TABLE alert_record (
  id          CHAR(32)     PRIMARY KEY,
  type        ENUM('SKU_OUT_OF_STOCK','SERVER_TRAFFIC_HIGH','RESOURCE_EXPIRING','NODE_FAILOVER','AGENT_OFFLINE') NOT NULL,
  severity    ENUM('INFO','WARN','ERROR') NOT NULL,
  target_type VARCHAR(32),
  target_id   VARCHAR(64),
  message     TEXT NOT NULL,
  resolved_at DATETIME,                            -- null = 未处理
  created_at  DATETIME     NOT NULL,
  INDEX idx_status (resolved_at, severity)

```

### `agent_task` — Agent 任务队列 (详见 05-Agent架构.md)

```sql
CREATE TABLE agent_task (

  server_id       CHAR(32)  NOT NULL,

  picked_at       DATETIME,
  result_payload  JSON,

```

## 3.2 老表加字段

### `resource_server` — 基础设施登记 + 域名 + lifecycle

```sql
ALTER TABLE resource_server
  ADD bandwidth_mbps           INT          COMMENT '运营商承诺峰值带宽 (账面, 仅展示)',
  ADD domain                   VARCHAR(128) UNIQUE COMMENT '线路机域名 (如 jp-01.nook.com), LIVE 前置条件',
  ADD cf_zone_id               VARCHAR(64)  COMMENT 'Cloudflare Zone ID',
  ADD cf_record_id             VARCHAR(64)  COMMENT 'Cloudflare DNS record ID',
  ADD cost_monthly_usd         DECIMAL(10,2),
  ADD billing_cycle_day        TINYINT      COMMENT '账单日 (1-28); 月度流量重置参考',
  ADD expires_at               DATE         COMMENT '服务器到期日',
  ADD max_concurrent_clients   INT          NOT NULL DEFAULT 50
                                            COMMENT '客户数硬上限 (allocator 必校验), admin UI 必填; 建议: 1C1G=50-100, 2C2G=200, 4C4G=500, 8C8G=1000',
  ADD lifecycle_state          ENUM('INSTALLING','READY','LIVE','RETIRED') NOT NULL DEFAULT 'INSTALLING',
  ADD last_heartbeat_at        DATETIME COMMENT 'agent 上次心跳时间',
  ADD temp_unhealthy           TINYINT NOT NULL DEFAULT 0 COMMENT '3-5min 心跳缺失暂时性不健康, allocator 跳过';
```

> **不卖产品也能用**: 资源管理 (xray 部署 / SSH 凭据) 在 resource_server 完成; 容量与售卖逻辑拆到下方 `resource_server_capacity`.

### `resource_server_capacity` — 服务器容量监控 (NIC 视角)

```sql
CREATE TABLE resource_server_capacity (
  server_id              CHAR(32)     PRIMARY KEY,
  monthly_traffic_gb     INT          COMMENT '机房配额 (NIC 视角); 0=不限',
  used_traffic_gb        INT          NOT NULL DEFAULT 0  COMMENT '当前周期已用 NIC 流量 (Agent 上报)',
  quota_reset_policy     ENUM('CALENDAR_MONTH','BILLING_CYCLE','FIXED') NOT NULL DEFAULT 'CALENDAR_MONTH'
                                      COMMENT 'CALENDAR_MONTH=每月 1 号重置; BILLING_CYCLE=按 billing_cycle_day; FIXED=永不',
  throttle_state         ENUM('NORMAL','THROTTLED') NOT NULL DEFAULT 'NORMAL'
                                      COMMENT 'used 90% → THROTTLED, allocator 跳过',
  created_at             DATETIME     NOT NULL,
  updated_at             DATETIME     NOT NULL,
  CONSTRAINT fk_capacity_server FOREIGN KEY (server_id) REFERENCES resource_server(id)
);
```

**v3 砍掉的字段**:
- `sold_traffic_gb` — 不再需要事前容量预约 (SKU 容量改为数 IP)
- `oversubscription_ratio` — 同上 (砍掉超售概念)

> **生命周期 vs 容量状态的两层划分**:
> - `resource_server.lifecycle_state`: 长周期状态机 (装机 → 上线 → 下架)
> - `resource_server_capacity.throttle_state`: 短周期容量状态 (仅 lifecycle=LIVE 期间有意义)
> - allocator 硬约束: `lifecycle_state=LIVE AND throttle_state=NORMAL` 才进候选池

> **READY → LIVE 上线前置条件** (admin 后台点 "上线" 时校验):
> 1. `domain` 字段已填
> 2. DNS 解析已生效 (后端 `dig +short <domain>` 验证)
> 3. `cf_zone_id` 已填
> 4. `monthly_traffic_gb` 已填到 capacity 表
> 5. `max_concurrent_clients` 已填

## 3.4 带宽策略 (不限速, 让用户跑满 IP 物理上限)

**v3 设计原则**: **不主动限速** (xray / dante 都不限). 用户能跑多快取决于:

1. **同 server 并发用户数** ← admin 控的 `max_concurrent_clients`
2. **线路机本身带宽** ← admin 进货控的 `bandwidth_mbps` (账面)
3. **落地 IP 机器带宽** ← admin 选 IP 机房控的
4. **国际链路质量** ← CN2 / IPLC vs 普通 IP

**两种运营策略 (admin 决策)**:

| 策略 | max_concurrent_clients | 每用户能跑 | 适合定位 |
|---|---|---|---|
| **低密度独享** | 1C2G 挂 10-20 用户 | 单用户 10-50 Mbps | 高端套餐 (¥150+) |
| **高密度低价** | 1C1G 挂 50-100 用户 | 单用户 1-5 Mbps | 闲鱼小作坊 (¥40-80) |

**v3 不在系统层做限速**, admin 通过 `max_concurrent_clients` 间接控制. 用户实际能跑多少 = `min(线路机带宽 / 并发数, 落地 IP 带宽, 国际链路上限)`.

**晚高峰 / 突发场景**: 单用户瞬间打满, 其他用户分摊带宽变慢. 这是已知的, 用户承诺时说"建议带宽 X Mbps" 而非"承诺 X Mbps".

**Sprint 5+ 可选**: 加 Linux `tc` 单用户限速 (per-user TC rule), 防止恶意刷流量影响他人. 起步不做.
