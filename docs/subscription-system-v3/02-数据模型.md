## 1. 通用约定

### 1.1 主键

业务表用 `id CHAR(32)` UUID (`@TableId(type = IdType.ASSIGN_UUID)`), MyBatis-Plus 在 INSERT 时生成 32 位无连字符 hex。

1:1 子表用上级表的 FK 作 PK (e.g., `resource_server_credential.server_id` PK = `resource_server.id`)。

字典表用业务码作 PK (`system_region.code = 'JP-TYO'`)。

### 1.2 时间字段

主表 (继承 `BaseEntity`): `created_at` / `updated_at`, 由 `MetaObjectHandlerImpl` 在 INSERT / UPDATE 时自动 fill。

1:1 子表不继承 `BaseEntity` (PK 不是 id), 字段手动 set 或在 Mapper `update` 时显式 `set updated_at = NOW()`。

### 1.3 软删除

`@TableLogic Integer deleted` 仅用于: `resource_server` / `member_user` / `system_user`。

1:1 子表不软删: 通过 join 主表 `deleted = 0` 过滤即可。

### 1.4 Update 策略

MP 全局 `update-strategy: NOT_NULL` — **PATCH 语义** (update 跳过 null 字段, 不能把字段从有值清成 null)。这是有意决策, 见 [07-决策 §3.2.4](./07-决策-排除-风险-Checklist.md)。

### 1.5 跨模块 API

业务模块拆 `nook-module-<x>-api` (对外契约: enums + DTO + Api interface) + `nook-module-<x>-server` (实现)。跨模块调用只走 `-api` 包, 不直接注入对方 Service / Mapper。

## 2. 资源 - server 系列 (线路机 + 落地机共用)

> **核心模型**: 线路机 (frontline) 和落地机 (landing) 都是 `resource_server` 的一行, 靠 `server_type` 区分。两者共用主表 + credential / billing / capacity / runtime 这 4 张通用子表; 各自再挂一张角色专属子表 (frontline → `resource_server_frontline`, landing → `resource_server_landing`)。

```
resource_server (server_type)
  ├─ resource_server_credential   (1:1, 通用: SSH)
  ├─ resource_server_billing      (1:1, 通用: 账面)
  ├─ resource_server_capacity     (1:1, 通用: NIC 流量 + 限速阈值)
  ├─ resource_server_runtime      (1:1, 通用: 心跳)
  ├─ resource_server_frontline    (1:1, frontline 专属: DNS)        ── + xray_server / xray_config
  └─ resource_server_landing      (1:1, landing 专属: socks5/dante/占用)
```

### 2.1 `resource_server` — 主表 (低频) ✅

```sql
CREATE TABLE resource_server (
  id              CHAR(32)     PRIMARY KEY,
  server_type     ENUM('frontline','landing') NOT NULL DEFAULT 'frontline'
                                                   COMMENT 'agent 角色: frontline=线路机 landing=落地机',
  name            VARCHAR(64)  NOT NULL            COMMENT '别名 (admin UI 显示)',
  lifecycle_state ENUM('INSTALLING','READY','LIVE','RETIRED') NOT NULL DEFAULT 'INSTALLING'
                                                   COMMENT '装机生命周期; 走 transition 接口流转',
  total_ip_count  INT          NOT NULL DEFAULT 0  COMMENT 'IP 总数 (展示用)',
  region          VARCHAR(64)                      COMMENT 'FK → system_region.code',
  remark          VARCHAR(512),
  agent_token     CHAR(64)     UNIQUE              COMMENT 'Agent 鉴权 token; 装机一次性 SHA256 生成, 永不更新',
  created_at      DATETIME     NOT NULL,
  updated_at      DATETIME     NOT NULL,
  deleted         TINYINT      NOT NULL DEFAULT 0,
  INDEX idx_server_lifecycle (lifecycle_state, deleted)
);
```

> **无外键约束** (FK 仅作注释表达关系)。Nook 项目规范: 不建 DB 外键, region 等引用一致性由业务 Validator 兜底。

**Lifecycle 状态机** (`ResourceServerLifecycleEnum`): `INSTALLING → READY → LIVE → RETIRED`。仅 `LIVE` 参与分配候选。

允许的转移 (`ResourceServerServiceImpl.ALLOWED_LIFECYCLE_TRANSITIONS`):
```
INSTALLING ↔ READY
READY      ↔ LIVE
READY → RETIRED / LIVE → RETIRED
RETIRED → LIVE (复活)
```

**转 LIVE 前置**: frontline 必须 `resource_server_frontline.domain` 非空 (`transitionLifecycle` 校验)。

> ⚠️ 现状: lifecycle **不会自动推进**, 全靠 `/admin/resource/server/transition-lifecycle` 手动切。xray / socks5 装机完成不会自动 INSTALLING → READY/LIVE (这块自动化在 roadmap)。落地机 SOCKS5 装机收尾时会 `→ LIVE` (`finalizeInstall`), 线路机不会。

### 2.2 `resource_server_credential` — SSH 凭据 (1:1, 通用) ✅

```sql
CREATE TABLE resource_server_credential (
  server_id                  CHAR(32)     PRIMARY KEY,
  ssh_port                   INT          NOT NULL DEFAULT 22,
  ssh_user                   VARCHAR(64),                 -- 默认 root
  ssh_password               VARCHAR(255),                -- 明文 (后台受信网络); update 留空=保留原值
  ssh_timeout_seconds        INT,                          -- 握手超时
  ssh_op_timeout_seconds     INT,                          -- 单命令超时
  ssh_upload_timeout_seconds INT,                          -- SCP 超时
  install_timeout_seconds    INT,                          -- 装机整体超时
  created_at                 DATETIME     NOT NULL,
  updated_at                 DATETIME     NOT NULL
);
```

> SSH 主机 = `resource_server` 的 IP (canonical, 落在凭据相关 VO 的 host 字段, DB 不单独冗余)。
> LIVE 后 host / ssh_port 硬锁 (`ResourceServerCredentialValidator`); 密码留空保留原值。

### 2.3 `resource_server_billing` — 账面 (1:1, 通用) ✅

```sql
CREATE TABLE resource_server_billing (
  server_id         CHAR(32)       PRIMARY KEY,
  idc_provider      VARCHAR(64)                  COMMENT '云厂商',
  cost_monthly_usd  DECIMAL(10,2),
  billing_cycle_day INT                          COMMENT '账单日 1-28',
  expires_at        DATE                         COMMENT '服务器到期日',
  created_at        DATETIME       NOT NULL,
  updated_at        DATETIME       NOT NULL
);
```

全是 admin 录入, 后端无业务读取参与决策。(注: 旧设计的 `bandwidth_mbps` 账面字段当前 DDL 已无, 限速账面统一在 capacity 表的限速字段。)

### 2.4 `resource_server_frontline` — 线路机 DNS (1:1, frontline 专属) ✅

```sql
CREATE TABLE resource_server_frontline (
  server_id     CHAR(32)     PRIMARY KEY,
  domain        VARCHAR(128) UNIQUE              COMMENT '线路机域名, LIVE 前置必填',
  cf_zone_id    VARCHAR(64)                      COMMENT 'Cloudflare Zone ID',
  cf_record_id  VARCHAR(64)                      COMMENT 'CF DNS Record ID',
  created_at    DATETIME     NOT NULL,
  updated_at    DATETIME     NOT NULL
);
```

转 LIVE 前 `domain` 必填。xray install 可通过 `cf_zone_id` + CF API token 自动加 A 记录。

### 2.5 `resource_server_landing` — 落地机扩展 (1:1, landing 专属) ✅

落地机的占用状态 + socks5 配置 + dante 装机事实**全在这一张表**, 没有独立 install 子表。

```sql
CREATE TABLE resource_server_landing (
  server_id             CHAR(32)     PRIMARY KEY,           -- FK → resource_server.id

  -- 占用状态机 (allocator 视角)
  status                ENUM('AVAILABLE','RESERVED','OCCUPIED','COOLING') NOT NULL DEFAULT 'AVAILABLE'
                                                            COMMENT 'IP 占用状态',
  occupied_by_member_id CHAR(32)                            COMMENT 'OCCUPIED 时填占用会员',
  occupied_at           DATETIME,
  cooling_until         DATETIME                            COMMENT 'COOLING 到期时刻',
  reserved_expires_at   DATETIME                            COMMENT 'RESERVED 预占截止',
  assign_count          INT          NOT NULL DEFAULT 0     COMMENT '历史分配次数',

  -- 身份 / 部署模式
  provision_mode        TINYINT      NOT NULL DEFAULT 2     COMMENT '1=自部署 dante 2=第三方现成 socks5',
  ip_type_id            CHAR(32)                            COMMENT 'FK → system_ip_type.id',

  -- SOCKS5 服务参数
  socks5_port           INT                                 COMMENT 'SOCKS5 监听端口',
  socks5_username       VARCHAR(64)                         COMMENT '明文存储 (待加密)',
  socks5_password       VARCHAR(255)                        COMMENT '明文存储 (待加密)',
  log_level             VARCHAR(128)                        COMMENT 'dante log 关键字组合 (空格分隔)',
  log_path              VARCHAR(255)                        COMMENT 'dante logoutput 路径',
  autostart_enabled     TINYINT      NOT NULL DEFAULT 1     COMMENT 'systemd 开机自启',
  firewall_enabled      TINYINT      NOT NULL DEFAULT 1     COMMENT '部署时配 UFW',
  firewall_allow_from   VARCHAR(64)                         COMMENT 'UFW allow CIDR; NULL=0.0.0.0/0',
  install_dir           VARCHAR(255)                        COMMENT 'dante 安装目录, 默认 /home/socks5',

  -- dante 装机事实 (装机完成后回写)
  dante_version         VARCHAR(32)                         COMMENT 'dpkg 探测的 dante 版本',
  conf_path             VARCHAR(255)                        COMMENT 'sockd.conf 绝对路径',
  pam_file              VARCHAR(255)                        COMMENT 'PAM 配置文件路径',
  pwd_file              VARCHAR(255)                        COMMENT 'htpasswd 密码文件路径',
  systemd_unit          VARCHAR(64)                         COMMENT 'systemd 服务名 (默认 danted)',
  log_rotate_enabled    TINYINT                             COMMENT '是否配过 logrotate',
  installed_at          DATETIME                            COMMENT '装机完成时间',
  last_dante_uptime     DATETIME                            COMMENT '探测到的 dante 启动时间',

  created_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

**占用状态机** (`ResourceServerLandingStatusEnum`, 仅 lifecycle=LIVE 期间有意义):
```
AVAILABLE → RESERVED (支付预占, Sprint 3 才用)
          → OCCUPIED (xray_client 绑定中)
          → COOLING  (revoke 后按 system_ip_type.cooling_minutes 冷却防 IP 信誉污染)
          → AVAILABLE
```

> ⚠️ **落地机无 `bandwidth_limit_mbps` 字段** — 落地机限速 (dante per-IP) 当前 DDL 未加, enforce 也未实现。详见 [06-Roadmap §限速](./06-Roadmap-待建.md)。

### 2.6 `resource_server_capacity` — NIC 流量监控 (1:1, 通用, 中频 5min) 🟡

```sql
CREATE TABLE resource_server_capacity (
  server_id            CHAR(32)     PRIMARY KEY,
  monthly_traffic_gb   INT                                  COMMENT '机房 NIC 月配额 GB (0/null=不限)',
  client_max_count     INT          NOT NULL DEFAULT 0      COMMENT '单 server 客户端数硬上限; 0=不限; allocator + xray inbound config 用',
  rx_bytes             BIGINT       NOT NULL DEFAULT 0      COMMENT '当周期下行累计 (Agent vnstat 上报)',
  tx_bytes             BIGINT       NOT NULL DEFAULT 0      COMMENT '当周期上行累计',
  used_traffic_bytes   BIGINT       NOT NULL DEFAULT 0      COMMENT '= rx + tx',
  quota_reset_policy   VARCHAR(32)  NOT NULL DEFAULT 'FIXED' COMMENT 'BILLING_CYCLE / FIXED',
  throttle_state       ENUM('NORMAL','THROTTLED') NOT NULL DEFAULT 'NORMAL'
                                                            COMMENT 'used 90% → THROTTLED',
  bandwidth_limit_mbps INT          NOT NULL DEFAULT 0      COMMENT '线路机出站接口限速 Mbps; 0=不限; 设计上 agent tc qdisc 落实',
  created_at           DATETIME     NOT NULL,
  updated_at           DATETIME     NOT NULL
);
```

**关键字段变化** (相对旧文档):
- `client_max_count` — 单 server 客户端硬上限, **从旧 `xray_node.touchdown_size` 改名并挪到本表**。xray reinstall 时校验不能缩到比当前活客户数小。
- `quota_reset_policy` — 旧设计的 `CALENDAR_MONTH` 已删, 只剩 `BILLING_CYCLE` / `FIXED` (`ResourceServerQuotaResetPolicyEnum`)。

**已实现**: Agent NIC 上报 → `AgentReportService.applyNicTraffic` → 覆盖 rx/tx/used_traffic_bytes。

**🟡 半成品** (DDL 有, 业务未实现, 详见 [06-Roadmap](./06-Roadmap-待建.md)):
- `throttle_state`: 只初始化 NORMAL, 无 Job 翻转 THROTTLED
- `bandwidth_limit_mbps`: 字段落表, 但**无任何代码读它推 tc 规则** (限速 enforce 未实现)
- `quota_reset_policy` 周期切换 (跨期归零) 未实现
- `monthly_traffic_gb` 已有 admin UI 编辑入口 (capacity 编辑 dialog)

### 2.7 `resource_server_runtime` — 心跳运行时 (1:1, 通用, 高频 1min) ✅

```sql
CREATE TABLE resource_server_runtime (
  server_id            CHAR(32)     PRIMARY KEY,
  last_heartbeat_at    DATETIME                             COMMENT 'agent 上次心跳; null=从未',
  temp_unhealthy       TINYINT      NOT NULL DEFAULT 0      COMMENT '心跳 3-5min 缺失暂时不健康',
  agent_version        VARCHAR(32)                          COMMENT 'agent 自报版本',
  last_agent_seen_ip   VARCHAR(64)                          COMMENT 'agent 源 IP',
  consecutive_miss     INT          NOT NULL DEFAULT 0      COMMENT '连续缺失次数; 心跳到达清零',
  updated_at           DATETIME     NOT NULL                -- 无 created_at, init 时即占位
);
```

**派生在线状态** (前端展示, 不入库, `AgentOnlineState.classify`):
- `< 60s` → ONLINE / `60-180s` → WARN / `180-300s` 或 temp_unhealthy=1 → TEMP_UNHEALTHY / `≥ 300s` → OFFLINE / `last_heartbeat_at=null` → NEVER

`AgentHeartbeatTimeoutJob` 每 1min 扫表, `consecutive_miss` 累加满 3 次置 `temp_unhealthy=1`。

### 4.3 `xray_client` — Xray 客户端 (1 client = 1 会员 1 落地 IP) ✅

```sql
CREATE TABLE xray_client (
  id              CHAR(32)     PRIMARY KEY,
  server_id       CHAR(32)     NOT NULL                     COMMENT '线路机 ID',
  ip_id           CHAR(32)     NOT NULL                     COMMENT '落地机 ID (= resource_server.id, server_type=landing)',
  member_user_id  CHAR(32)     NOT NULL,
  client_uuid     VARCHAR(64)  NOT NULL                     COMMENT 'vless/vmess UUID / trojan 密码',
  client_email    VARCHAR(128) NOT NULL                     COMMENT '格式 member_{会员ID}_{IP ID}',
  status          TINYINT      NOT NULL DEFAULT 1           COMMENT '1=运行 2=已停 3=待同步 4=远端已不存在',
  last_synced_at  DATETIME                                   COMMENT '与远端 xray 对账时间',
  created_at      DATETIME     NOT NULL,
  updated_at      DATETIME     NOT NULL
);
```

> `ip_id` 指向 `resource_server.id` (server_type=landing 的落地机)。v3 "1 IP = 1 客户" 由业务保证 (provision 时占用 landing CAS, revoke 时归还)。
> `status` 已有 `XrayClientStatusEnum` (RUNNING=1 / STOPPED=2 / PENDING_SYNC=3 / REMOTE_ABSENT=4)。

**操作** (全走 op 框架, 见 [03-业务核心](./03-业务核心-算法与流程.md)): provision (占落地机 + insert + 远端 adu/adr/ado) / revoke (远端清理 + 归还落地机 cooling) / rotate (换 UUID) / sync / replay。

### 4.4 `xray_client_traffic` — Xray 累计流量样本 ✅

```sql
CREATE TABLE xray_client_traffic (
  id                    VARCHAR(64)  PRIMARY KEY,
  server_id             VARCHAR(64)  NOT NULL                COMMENT '冗余, 加速按 server sweep',
  client_id             VARCHAR(64)  NOT NULL                COMMENT 'xray_client.id (1:1)',
  uplink_bytes          BIGINT       NOT NULL DEFAULT 0      COMMENT '累计上行, 跨 xray 重启不丢',
  downlink_bytes        BIGINT       NOT NULL DEFAULT 0      COMMENT '累计下行',
  last_counter_uplink   BIGINT                               COMMENT '上次采样 xray counter 值 (上行); 算增量用; NULL=未建基线',
  last_counter_downlink BIGINT,
  last_sampled_at       DATETIME,
  created_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

**采样**: agent 上报 xray stats 当前累计 → `delta = current - last_counter_*` 累加。跨 xray 重启 (counter 归零) delta 为负时不累加 (基线重置)。

## 8. 设计原则

### 8.1 双层状态机 (lifecycle vs 业务状态)

| 表 | 维度 A (低频, admin 视角) | 维度 B (高频, 业务视角) |
|---|---|---|
| `resource_server` | `lifecycle_state` (装机) | `resource_server_capacity.throttle_state` (容量, 🟡 未启用) |
| `resource_server_landing` | (随主表 lifecycle) | `status` (占用) |

**通用规则**: 仅当 lifecycle_state=LIVE 时, 高频维度才被业务读取。admin 下架 (→ RETIRED) 不影响在跑客户, 分配器自然停止派单。

### 8.2 子表分层 (按更新频率 / 职责)

`resource_server` 拆主表 + 6 张子表:

| 表 | 更新频率 | 拆分理由 |
|---|---|---|
| `resource_server` (低频) | 装机一次 + 偶尔改 | 冷数据 |
| `resource_server_capacity` (中频 5min) | NIC 上报 | 高频写隔离 |
| `resource_server_runtime` (高频 1min) | 心跳 | 心跳 50w+/年, 隔离避免污染主表 buffer pool |
| credential / billing / frontline / landing | 偶尔改 | 职责单一 (SSH / 账面 / DNS / socks5 互不相关) |

### 8.3 表前缀

| 模块 | 前缀 | 表 |
|---|---|---|
| 资源 | `resource_*` | resource_server* |
| 字典 | `system_*` | system_region / system_ip_type / system_user* |
| 会员 | `member_*` | member_user |
| Xray | `xray_*` | xray_server / xray_config / xray_client / xray_client_traffic |
| Agent | `agent_*` | agent_runtime_config / agent_task |
| 编排 | `op_*` | op_config / op_log |
| (待建) 交易 | `trade_*` | trade_plan / trade_plan_resource / trade_subscription / trade_order / trade_order_item, 见 [06-Roadmap](./06-Roadmap-待建.md) |
| (待建) 告警 | `alert_*` | alert_record (归属待定, 非 trade) |
