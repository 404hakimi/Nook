## 一、产品形态

Nook 卖**独享 IP 代理服务**, **运营模型类比独享公寓出租**:
- **公寓 (落地 IP)** — 用户独享一套, 14 天保留期
- **门禁 (线路机)** — 多套公寓共用一个入口, 用户的 vmess 流量从这里进
- **套餐 SKU** — 商品 = 一组门禁 + 一组公寓 + 流量限额 + 月租

**心智模型**: admin **基于现有资源** 决定上架哪些 SKU (自下而上). 公寓卖光 → SKU 自动售罄, 加货 → 自动上架.

- **客户**: 跨境电商 / LLM 开发者 / 远程办公; 对"住宅 IP / ISP IP / 干净 IP"敏感
- **商品**: 由 admin 预定义并绑定具体资源; 不暴露用户自定义构建
- **价位带**: 闲鱼小作坊级 SaaS, 单 IP 套餐 ¥40-¥300/月
- **客户感知**:
  - 每个 sub 一个**独立子域名** (e.g., `jp-alice-001.nook.com`), 写在 vmess URL 里
  - 用户级聚合订阅 URL (member.sub_token) 返回该用户所有活跃 sub 的 vmess 节点
  - 切换线路机时 DNS 改指向, 用户客户端不刷订阅也能自动续上 (5-15 min DNS 缓存过期)
  - 14 天 IP 保留期: 套餐过期后, IP 仍归用户 14 天, 续费能拿回原 IP

### SKU 与资源的关系

```
[SKU: JP 住宅 100GB]
  ↓ plan_sku_resource 关联表 (详见 02-数据模型.md)
  ├─ 门禁池: [server-jp-01, server-jp-02, server-jp-03]
  └─ 公寓池: [ip-jp-1, ip-jp-2, ..., ip-jp-10]
       ↓
  容量 = 池里 AVAILABLE 公寓数 (e.g., 10)
  已售 = 已被占用的公寓数
  剩余 = 容量 - 已售 → 显示在商品页

[卖出一个套餐]
  从公寓池选一个空闲 IP + 从门禁池选客户数最少的 server
  分配给用户, 公寓状态: 空闲 → 已占用 (该用户)
  
[售罄]
  剩余 = 0 → 商品页 "已售罄, 不可购买"
  admin 加 IP 进池 → 剩余 +N → 自动上架
```

### 业务流程概览

1. **admin 进货** → 上架 SKU 时绑定门禁池 + 公寓池 → SKU 容量自动按公寓池算
2. **用户购买 SKU** → allocator 从池里挑 1 公寓 + 1 门禁 → backend 派任务给门禁 agent → agent 调本地 xray API 添加用户配置 → 同时调 Cloudflare API 创建独享子域名
3. **用户使用** → 客户端连子域名 → DNS 解析到门禁 IP → xray 认证 → 路由到公寓 (用户独享) → 出站
4. **故障 / 流量耗尽** → 池内调度选其他门禁 + 改子域名 DNS → 用户 5-15min 自动恢复
5. **运行期** → 每台机器跑 agent → 主动 push 流量 / 心跳 / 状态 → backend 收数据 + 派任务
